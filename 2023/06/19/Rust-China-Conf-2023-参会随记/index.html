    Rust China Conf 2023 参会随记

    Yuan Gao
    2023-06-19

    周六周日去上海参加了Rust China Conf 2023，以下是我听到的印象比较深的话题。因为对Rust实际应用较少，之后PPT上传之后还要对其中的一些议题进行进一步的研究。

    Borsh - 安全高效的二进制序列化

    Borsh是NEAR开发的多语言的二进制序列化框架。它的特点是不保存任何类型信息，只在假设类型正确的情况下保存必要的数据信息，使得序列化后的二进制数据相比用Json来序列化体积小且更高效。通过与Rust常见序列化框架bincode，cbor，speedy比较，Borsh在输入大小增加情况下，有更好的序列化和反序列化表现，相关Benchmark在官方Github库有一个部分专门介绍。

    演讲结束有两个问题，一个是关于Borsh和ProtoBuf这样传统序列化框架的区别，另外一个是省略类型信息是否降低了安全性。演讲者提出ProtoBuf更多会用在RPC(Remote Procedure Call)场景，并且ProtoBuf会有校验过程。Borsh在设计之初是假定序列化后的二进制数据是独占访问，不可能被篡改，同时Borsh是应用在区块链场景，有安全性的保证。

    项目Github链接，截至目前Borsh支持9种语言的二进制序列化。

    Rust原子操作的高性能实践

    这场演讲并没有介绍太多的前沿知识，更多的是介绍一些计算机基础知识，如

    • 原子性：有两种方式Compare and swap和Fetch and modify
    • 什么是指令重排
    • 如何获取高性能？无锁编程，减少上下文切换，同时为了减少contention，要引入Exponential Backoff Retry
    • C++/Rust的内存顺序：Sequential Consistency，Acquire，Release，AcqRel，Relaxed，Consume，Rust中是前四种，这篇官方文档有详细介绍
      • 对于AcqRel：
        • A write-release guarantees that all preceding code completes before the releasing write
        • A read-acquire guarantees that all following code starts after the acquiring read
        • 意思是说对于一个acquire访问，所有在它之后的操作，始终保持在它之后，不会提前执行，但对于acquire之前的操作可能会被重排到acquire之后。同理对于release访问，所有在它之前的操作，始终保持在它之前，但对于release之后的操作可能会被重排到release之前

    CeresDB的Rust生产实践

    CeresDB是一个分布式，支持云原生的时序数据库，可以用来存储和管理时间序列数据。目标是解决时间线高基数问题，并提供原生分布式方案，适用于物联网IoT、运维监控、金融分析等行业场景。

    Apache Ballista

    Apache Ballista是一个Rust实现的基于Apache Arrow和DataFusion实现的分布式的SQL查询引擎，项目Github链接。

    与Apache Spark 类似，它实现了一个分布式执行模型，支持基于DAG的任务调度和容错。它还支持不同工作节点之间的数据交换。关键的区别是，混洗数据是以Arrow格式存在的，可以通过使用Flight协议进行高效交换，实现零拷贝。Ballista支持许多云对象存储，如 HDFS、S3、Azure。与Snowflake类似，为了加速查询性能，Ballista也支持在本地磁盘上的数据缓存和缓存感知的任务调度。
